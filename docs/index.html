Your agent loads the junk drawer too.

One project needs GitHub and Postgres. Another needs Playwright. A third needs a docs server. After a few weeks, your MCP configs look like an old startup disk: useful tools mixed with trial installs, duplicates, stale packages, and permissions you would not add today.

  • Audit the whole machine. Claude, Codex, Cursor, Windsurf, Copilot, VS Code, Cline, Roo Code, Continue, Zed, and project `.mcp.json` files.
  • Clean obvious clutter. Duplicate servers, dead commands, missing paths, archived repos, and packages stuck on `latest`.
  • Reclaim context. Rank servers by estimated tool count so heavy MCPs stay out of projects that do not need them.
  • Find old installs. `--track-usage` builds a local ledger of MCPs that keep hanging around scan after scan.

What it finds before your agent trips over it.

Abandoned servers

Checks npm metadata and GitHub activity so archived or quiet MCP projects are easy to remove.

Context hogs

Ranks heavy servers like GitHub, browser automation, Slack, Jira, and filesystem tools by estimated tool load.

Risky configs

Flags broad filesystem mounts, inline secrets, broken env refs, dead paths, duplicate names, and unpinned packages.

Start with one scan. Delete nothing until you approve it.

The first command gives you a redacted local cleanup report. If you want changes, MCP Doctor creates named patch plans with backups. You decide what gets removed.

npx @inferensys/mcp-doctor scan --workspace .
npx @inferensys/mcp-doctor scan --registry --track-usage
npx @inferensys/mcp-doctor report --format html
npx @inferensys/mcp-doctor server
